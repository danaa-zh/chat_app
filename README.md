# AlmaTalk 💬

A real-time mobile chat application built with Flutter and Firebase, designed for seamless communication with a modern, clean UI. AlmaTalk supports a full friend system, multilingual interface, and live messaging — all backed by Cloud Firestore.

## Features

### Authentication
- Email & password sign-in and registration
- Google Sign-In
- Forgot password & change password flows
- Persistent session via Firebase Auth state stream

### Messaging
- Real-time one-on-one chat powered by Cloud Firestore
- Message read receipts and delivery status
- Unread message counters per conversation
- Message editing and reply-to support
- Per-user soft delete (chat history can be cleared without affecting the other participant)

### Social / Friend System
- Discover users via the Find People screen
- Send, receive, accept, and reject friend requests (with separate Sent / Received tabs)
- Friends list with live online/offline presence indicators
- Block support baked into the friendship model

### Notifications
- In-app notification centre for friend request events and new messages
- Mark-as-read and delete individual notifications

### Profile
- View and edit display name and avatar
- Change password from within the app
- Online status automatically updated on sign-in and sign-out

### UI & UX
- Custom theme built on Poppins (Google Fonts) with an olive-green palette (`#9CAD75`)
- Adaptive splash screen and launcher icon (Android)
- Fully localised in **English**, **Kazakh**, and **Russian**
- Language switcher persisted locally via GetStorage

## Architecture

AlmaTalk follows a **feature-first layered architecture** to keep concerns clearly separated and the codebase easy to scale.

```
Presentation  →  Controllers (GetX)  →  Repositories  →  Services  →  Firebase
```

- **Presentation** — stateless `View` widgets consuming reactive GetX observables
- **Controllers** — business logic, bound to routes via GetX `Bindings`
- **Repositories** — thin abstraction over service calls, one per domain (Auth, Chat, Friends, etc.)
- **Services** — direct Firebase SDK calls (`AuthService`, `FirestoreService`)
- **Models** — immutable data classes with `toMap` / `fromMap` serialisation

## Getting Started

### Prerequisites

- Flutter SDK `>=3.11.0`
- Dart SDK `>=3.0.0`
- A Firebase project (see [Firebase Setup](#firebase-setup))
- Android Studio / VS Code with Flutter & Dart plugins

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/your-username/alma-talk.git
cd alma-talk/chat_app

# 2. Install dependencies
flutter pub get

# 3. Generate splash screen and launcher icons
dart run flutter_native_splash:create
dart run flutter_launcher_icons

# 4. Create your .env file (see Environment Variables)
cp .env.example .env

# 5. Run the app
flutter run
```

## Environment Variables

Create a `.env` file in the project root. The file is included in `flutter` assets automatically:

```env
# .env
SOME_API_KEY=your_value_here
```

> **Note:** Never commit `.env` to version control. Add it to `.gitignore`.

## Firebase Setup

1. Create a project at [console.firebase.google.com](https://console.firebase.google.com).
2. Enable **Authentication** → Email/Password and Google providers.
3. Create a **Cloud Firestore** database (start in test mode, then apply rules).
4. Install the FlutterFire CLI and run:

```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

This regenerates `lib/firebase_options.dart` for your project.

### Firestore Collections

| Collection | Purpose |
|---|---|
| `users` | User profiles, online status, lastSeen |
| `chats` | Chat metadata (participants, lastMessage, unreadCount) |
| `chats/{chatId}/messages` | Individual messages per conversation |
| `friendRequests` | Pending / accepted / rejected requests |
| `friendships` | Confirmed friend pairs (with block support) |
| `notifications` | Per-user in-app notifications |

## Localization

AlmaTalk is fully internationalised using Flutter's official `intl` tooling with ARB files.

**Supported locales:**

| Code | Language |
|---|---|
| `en` | English |
| `kk` | Kazakh (Қазақша) |
| `ru` | Russian (Русский) |

The active language is persisted locally via `GetStorage` and can be switched at runtime through the `LanguageController`.

To add a new locale:
1. Create `lib/l10n/app_<locale>.arb` with translated strings.
2. Add the `Locale('<locale>')` to `supportedLocales` in `main.dart`.
3. Run `flutter gen-l10n` to regenerate the `generated/` classes.
